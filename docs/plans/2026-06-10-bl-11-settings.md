# BL-11 — Settings loading, validation & schema

## Context

BL-11 adds the settings foundation that all later behavior modules depend on.
`src/settings.ts` is currently a stub that exports nothing. This plan:
1. Creates `settings-schema.json` with the v1 settings and per-action shortcut bindings.
2. Replaces the stub with a `MintangleSettings` class wrapping Cinnamon's
   `imports.ui.settings.ExtensionSettings`.
3. Updates the build script to copy the schema into `build/`.
4. Extends `types/cinnamon.d.ts` with a minimal `ExtensionSettings` declaration.

## Cinnamon Settings Mechanism

- `ExtensionSettings(bindObject, uuid)` reads `settings-schema.json` from the
  installed extension directory and stores user values in
  `~/.cinnamon/configs/<uuid>/`.
- `getValue(key)` / `setValue(key, value)` for direct reads/writes.
- `connect('changed::<key>', cb)` for change signals; `disconnect(id)` removes them.
- `finalize()` disconnects all bindings and signals — call on disable.

## 1. `settings-schema.json`

Located in the repo root (alongside `metadata.json`). Copied verbatim into `build/` by the build script.

**Boolean settings** — type `switch`, all default `true`:
- `enable-cycling` — Enable repeated command cycling
- `enable-half-cycling` — Enable half position cycling
- `enable-third-cycling` — Enable third position cycling
- `enable-fourth-cycling` — Enable fourth position cycling
- `enable-corner-cycling` — Enable corner position cycling
- `enable-display-cycling` — Enable display cycling

**Integer setting** — type `spinbutton`:
- `repeat-timeout` — default `1500`, min `100`, max `10000`, step `100`, units `"ms"`

**Shortcut bindings** — type `keybinding`, 26 entries named `shortcut-{action-id}`:

Each uses the GDK accelerator string from `DEFAULT_SHORTCUTS` in `src/constants.ts`.
Key names follow the pattern `shortcut-left-half`, `shortcut-next-display`, etc.
The action-id is already kebab-case so it's used verbatim after the prefix.

Example entries:
```json
"shortcut-left-half":  { "type": "keybinding", "default": "<Primary><Super>Left",  ... }
"shortcut-next-display": { "type": "keybinding", "default": "<Primary><Super><Alt>Right", ... }
```

## 2. `src/settings.ts`

Exports:
- Setting key constants (`KEY_ENABLE_CYCLING`, `KEY_REPEAT_TIMEOUT`, …) — used by BL-12/BL-09 to connect change listeners without hardcoding strings.
- `shortcutKey(action: ActionId): string` — returns `shortcut-${action}`.
- `REPEAT_TIMEOUT_MIN`, `REPEAT_TIMEOUT_MAX` — bounds used for coercion.
- `class MintangleSettings`

### `MintangleSettings` class

**Constructor:** `(bindObject: object, uuid: string)`  
Calls `new imports.ui.settings.ExtensionSettings(bindObject, uuid)` wrapped in
try/catch. On failure, logs with `logError` and sets `_settings = null` so all
getters fall back to defaults.

**Typed getters:**

| Method | Returns | Fallback |
|--------|---------|---------|
| `enableCycling()` | `boolean` | `true` |
| `repeatTimeout()` | `number` (clamped to [100, 10000]) | `1500` |
| `enableHalfCycling()` | `boolean` | `true` |
| `enableThirdCycling()` | `boolean` | `true` |
| `enableFourthCycling()` | `boolean` | `true` |
| `enableCornerCycling()` | `boolean` | `true` |
| `enableDisplayCycling()` | `boolean` | `true` |
| `shortcut(action: ActionId)` | `string` | `DEFAULT_SHORTCUTS[action]` |

**Private helpers:**
- `_getBool(key, fallback)` — `getValue(key)`, type-checks to boolean.
- `_getInt(key, fallback, min, max)` — `getValue(key)`, `Number()`, clamps via `Math.max/min`.
- `_getString(key, fallback)` — `getValue(key)`, validates non-empty string.

All helpers catch exceptions and return the fallback.

**Change listeners:**
- `connect(key: string, callback: () => void): number` — wraps
  `_settings.connect('changed::' + key, callback)`, tracks returned ID in
  `_connections[]`. Returns `0` if `_settings` is null.
- `disconnect(id: number): void` — calls `_settings.disconnect(id)` and removes
  from `_connections[]`. Skips if id is `0` or `_settings` is null.

**Cleanup:**
- `destroy(): void` — calls `_settings.finalize()` (disconnects all at once),
  nulls `_settings`, clears `_connections`.

## 4. `types/cinnamon.d.ts`

Add a typed declaration for `imports.ui.settings` so TypeScript understands
`ExtensionSettings` without losing type safety:

```typescript
interface CinnamonExtensionSettings {
  getValue(key: string): unknown;
  setValue(key: string, value: unknown): void;
  connect(signal: string, callback: (...args: any[]) => void): number;
  disconnect(id: number): void;
  finalize(): void;
}

// narrow imports.ui.settings
```

Extend the existing `imports` declaration's `ui` namespace to include:
```typescript
ui: {
  settings: {
    ExtensionSettings: new (bindObject: object, uuid: string) => CinnamonExtensionSettings;
  };
  [key: string]: any;
};
```
