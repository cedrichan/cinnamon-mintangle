# BL-16 — Fail-safe handling & error notifications

**Date:** 2026-06-11  
**Status:** Complete

## Goal

Harden `src/actions.ts` and related types against all documented failure modes.
Provide clear developer logging. User-visible notifications are deferred to BL-15
(no notification settings exist in PRODUCT.md yet).

## Scope

### What's already correct (no changes needed)
- `keybindings.ts`: registration failures are caught, logged, and stored in `_errors`. ✓
- `_handleRestore`: try/catch around `move_resize_frame`. ✓
- `_handleDisplayMovement`: try/catch around work-area fetch and move. ✓

### Changes

#### `types/cinnamon.d.ts`
- Add `is_resizable(): boolean` to `MetaWindow`.
- Add `get_monitor_geometry(monitorIndex: number): MetaRectangle` to `MetaDisplay`
  (raw monitor bounds, used as work-area fallback).

#### `src/actions.ts` — `dispatchAction` hardening
1. **No focused window**: add `log()` before silent return.
2. **Unsupported window type**: add `log()` when skipping desktop/dock.
3. **Non-resizable window**: check `win.is_resizable()` after window-type check;
   log and return if false.
4. **Work-area unavailable**: wrap `get_work_area_current_monitor()` in try/catch;
   fall back to `global.display.get_monitor_geometry(win.get_monitor())` with a
   `log()` warning. If both fail, log and return.

### Out of scope for this task
- User-visible notifications (deferred to BL-15).
- Keybinding conflict UI (deferred to BL-15).
