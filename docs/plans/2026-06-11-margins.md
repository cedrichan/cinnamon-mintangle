# Plan: Margin / Gap Setting with Collapsing

**Date:** 2026-06-11

## Goal

Add a user-configurable gap between tiled windows and between windows and screen
edges. Adjacent tiled windows should have the same visual gap as the gap between
windows and screen edges (margin collapsing).

## Behavior

- Each window placement respects a uniform `margin` value (pixels).
- Gap from screen edge to window = `margin` (provided by `applyMargins`).
- Gap between two adjacent tiled windows = `margin` (margin collapsing).
- When `margin = 0` (default), behavior is identical to today.
- Maximize, almost-maximize, center, and center-prominently use only the
  screen-edge margin (from `applyMargins`); no inter-tile gap applies since
  they are not tiled relative to another window.

## Settings

New setting: `margin` (type: `scale` / slider).

| Setting | Type    | Default | Min | Max |
| ------- | ------- | ------- | --- | --- |
| margin  | integer | 0       | 0   | 32  |

Added as a slider to `settings-schema.json` under a new "Layout" header.

## Implementation

### 1. `src/geometry.ts`

Add a private `computeSlot(start, totalSize, margin, total, index, span)` helper.

The helper computes the position and size of one tile slot in an axis-split
layout. It uses boundary-point math so adjacent slots share the same pixel edge
(max 1px rounding difference, same as the existing tiling guarantee).

Derivation for N equal tiles with (N−1) inter-tile gaps of size `margin`:
  effectiveSize = totalSize − (N−1)×margin
  boundary_k    = Math.round(effectiveSize × k / N)
  slot_pos      = start + boundary_index + index×margin
  slot_size     = boundary_(index+span) − boundary_index + (span−1)×margin

When margin = 0, degenerates to the current boundary-point formulas exactly.

Update every tiling function (halves, corners, thirds, two-thirds, fourths,
three-fourths) to use `computeSlot`.

### 2. `src/settings.ts`

Add:
  - `KEY_MARGIN = 'margin'`
  - `MARGIN_DEFAULT = 0`, `MARGIN_MIN = 0`, `MARGIN_MAX = 32`
  - `margin()` typed getter (integer, clamped)

### 3. `settings-schema.json`

Add `layout-header` and `margin` (type: `scale`) entries before the cycling
section.

### 4. `src/actions.ts`

Replace `const margin = 0` with `const margin = settings.margin()`.

### 5. `PRODUCT.md`

Extend the Settings table to include the margin setting.

## Acceptance Tests

1. With margin = 0, all layouts produce the same coordinates as before.
2. With margin = 10, left-half and right-half have a 10px gap between them.
3. With margin = 10, all tiled windows have a 10px gap from the screen edge.
4. Corners tile with 10px gap between all four cells.
5. Thirds tile with 10px gap between each third.
6. Maximize fills the work area minus the 10px margin on each side.
7. The margin slider is visible in the settings UI.
8. Changing the slider value takes effect on the next shortcut press.
